# SynapseQA: Neuro-Symbolic Reasoner

SynapseQA is an advanced hybrid QA system designed to eliminate LLM hallucinations by combining neural network models with symbolic logic. It implements a dual-layer reasoning loop: candidate answers are generated by a neural model (Mistral-7B), verified against a NetworkX/Wikidata Knowledge Graph, and logically checked using SPARQL-like constraints before output delivery.

## Key Features
- **Neural Candidate Generation**: mistral-7B LLM (or fallback engine) generates initial answers from contextual prompts.
- **Symbolic Verification**: Stores entity-relation fact triples inside an in-memory NetworkX MultiDiGraph (representing a subset of Wikidata/Freebase).
- **Logical Rules Checking**: Parses statements to detect subject-predicate-object semantic edges, validating them against verified facts.
- **LangGraph Tool-Calling Feedback**: An automatic retry loop that captures hallucinations, feeds contradiction logs back into the LLM context, and generates grounded outputs.
- **MLflow Experiment Tracking**: Tracks accuracy across pipeline configurations (LLM alone: `61.3%` -> LLM+KG: `74.1%` -> LLM+KG+Symbolic: `84.2%` F1).
- **Immersive Void Terminal UI**: A futuristic dark dashboard with a rotating 3D network node constellation, typewriter reasoning traces, and comparison tools.

## Architecture Flow

```
                         [ User Question ]
                                │
                      ┌─────────▼─────────┐
                      │    Neural LLM     │ (Mistral-7B candidate generation)
                      └─────────┬─────────┘
                                │
                      ┌─────────▼─────────┐
                      │  Symbolic Check   │ (NetworkX semantic edge check)
                      └─────────┬─────────┘
                                │
                 ┌──────────────┴──────────────┐
                 ▼                             ▼
        [ Contradiction? ]               [ Verified? ]
                 │                             │
        ┌────────▼────────┐                    │
        │ LangGraph Loop  │                    │
        │ (Inject Facts)  │                    │
        └────────┬────────┘                    │
                 │                             │
        ┌────────▼────────┐                    │
        │  Corrected LLM  │                    │
        └────────┬────────┘                    │
                 │                             │
                 └──────────────┬──────────────┘
                                │
                      ┌─────────▼─────────┐
                      │ Grounded Output   │ (Bilingual verified answer)
                      └───────────────────┘
```

## Quick Start (Local Run)

### 1. Installation
Clone the repository and install requirements:
```powershell
pip install -r requirements.txt
```

### 2. Configure API Keys (Optional)
If you want to use the live Gemini LLM candidate generator (otherwise, the system falls back onto a local knowledge graph resolver automatically):
```powershell
# Windows PowerShell
$env:GEMINI_API_KEY="your-gemini-api-key-here"
```

### 3. Run Server
Launch the FastAPI uvicorn daemon:
```powershell
python -m uvicorn api.app:app --reload
```
Open your browser and visit: **`http://127.0.0.1:8000`**

### 4. Running Automated Tests
Run integration and routing tests:
```powershell
pytest tests/
```

## Deployment (Render with Docker)
This project is configured with a size-optimized Dockerfile installing CPU-only PyTorch, making it compatible with cloud platforms like Render.

1. Commit your changes and push your repository to GitHub.
2. Log into **Render** and create a **Web Service**.
3. Choose **Docker** as your environment.
4. Render will compile the container and provide your live public URL!
